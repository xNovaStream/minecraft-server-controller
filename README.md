# Управление Minecraft-сервером с автоматическим отключением

Скрипты для автоматического управления Minecraft-сервером на платформе **clo.ru**.  
Мониторинг отслеживает количество игроков и при отсутствии активности в течение часа останавливает сервер для экономии ресурсов. Управление осуществляется через веб-интерфейс.

---

## 📦 Структура проекта

```
.
├── config.py          # Конфигурация, API-функции, работа с файлом-флагом
├── monitor.py         # Основной цикл мониторинга
├── web.py             # Веб-интерфейс (Flask)
├── templates/
│   └── index.html     # HTML-страница с кнопками и статусами

```

## 🚀 Как это работает

1. **Мониторинг** (`monitor.py`) запущен на сервере, который имеет доступ к Minecraft-серверу (по сети). Он раз в минуту проверяет онлайн игроков через `mcstatus`.
2. Если игроков нет, запускается таймер бездействия (по умолчанию 1 час). Если за это время никто не зашёл – сервер останавливается через API clo.ru. Мониторинг автоматически отключается.
3. **Веб-интерфейс** (`web.py`) предоставляет удобное управление:
   - Запустить сервер (и включить мониторинг после успешного запуска)
   - Включить/отключить мониторинг вручную
   - Просматривать текущий статус сервера и мониторинга
4. Веб-интерфейс доступен по адресу `http://<ваш_сервер>:5000` (порт по умолчанию).
5. При остановке сервера игроки не могут подключиться. После запуска через веб – сервер снова доступен.

## ⚙️ Настройка

### 1. Установите зависимости
```bash
pip install mcstatus flask requests
```

### 2. Заполните конфигурацию в `config.py`
```python
CLOUD_API_TOKEN = "ваш_токен_доступа_к_API_clo.ru"
MINECRAFT_SERVER_ID = "id_сервера_майнкрафт"
MINECRAFT_HOST = "ip_или_хост_майнкрафт"   # адрес для проверки онлайн
MINECRAFT_PORT = 25565
IDLE_TIMEOUT = 3600          # время без игроков до остановки (сек)
CHECK_INTERVAL_RUNNING = 60  # интервал проверки при работающем сервере
CHECK_INTERVAL_STOPPED = 30  # интервал проверки, когда мониторинг отключён (проверяет не включен ли мониторинг)
MONITOR_DISABLE_FILE = "/tmp/minecraft_monitor_disabled"  # путь к файлу-флагу
```

### 3. Запуск
```bash
python3 monitor.py   # в фоне (screen, systemd, nohup)
python3 web.py       # в фоне или через systemd
```

Веб-интерфейс будет доступен по адресу: `http://<ваш_сервер>:5000`

## 🧩 Логика работы

### Мониторинг (`monitor.py`)
- Бесконечный цикл, каждую минуту (или чаще) проверяет:
  1. Не отключён ли мониторинг флагом (файл существует)
  2. Подключается к Minecraft-серверу и получает онлайн
  3. Если онлайн == 0 – запускает таймер
  4. Если онлайн > 0 – сбрасывает таймер
  5. Если таймер превысил `IDLE_TIMEOUT` – останавливает сервер через API и отключает мониторинг

### Веб-интерфейс (`web.py`)
**Эндпоинты:**

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` | Главная страница (HTML) |
| POST | `/start-minecraft` | Запускает сервер, ожидает статус `ACTIVE`, включает мониторинг |
| POST | `/monitoring` | Включает мониторинг (удаляет файл-флаг) |
| DELETE | `/monitoring` | Отключает мониторинг (создаёт файл-флаг) |
| GET | `/monitoring/status` | Возвращает статус мониторинга: `{"enabled": true/false}` |
| GET | `/server/status` | Возвращает статус сервера из API: `{"status": "ACTIVE"}` и т.д. |

### Файл-флаг мониторинга
- Существование файла `MONITOR_DISABLE_FILE` означает, что мониторинг **отключён**.
- При включении мониторинга файл удаляется.
- Монитор проверяет его наличие на каждой итерации.

### Статусы сервера (из документации clo.ru)
- `BUILDING` – создаётся
- `ACTIVE` – активен, работает
- `STARTING` – запускается
- `STOPPING` – останавливается
- `STOPPED` – остановлен
- `REBOOTING` – перезагружается
- `RESIZING` – изменение ресурсов
- `BLOCKED` – заблокирован
- `DELETING` – удаляется
- `UNKNOWN` – неизвестно
- `ERROR` – ошибка

Кнопка **«Запустить»** активна **только** когда статус `STOPPED`.

## 🛠️ Управление и диагностика

- **Логи** выводятся в консоль (при запуске `monitor.py` и `web.py`). В веб-интерфейсе есть блок для отображения последних действий.
- При необходимости вручную отключить мониторинг можно через веб-кнопку или создав файл:
  ```bash
  touch /tmp/minecraft_monitor_disabled
  ```
  А включить – удалив его:
  ```bash
  rm /tmp/minecraft_monitor_disabled
  ```

## ⚠️ Примечания

- Скрипт `monitor.py` должен иметь доступ к Minecraft-серверу (по сети) для проверки онлайна.
- Для работы `mcstatus` необходим открытый порт `25565` на сервере с Minecraft.
- Рекомендуется настроить автоматический перезапуск скриптов через `systemd` или `supervisor`.
